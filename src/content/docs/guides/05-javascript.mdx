Javascript es un lenguaje de programacion que se utiliza para crear sitios web interactivos, sus caracteristicas:
- **Lenguaje de programacion interpretado**: significa que el codigo se ejecuta linea por linea, no se compila en ningun momento.
- **Se ejecuta solamente en el navegador**: es decir, que el codigo Javascript se ejecuta en el navegador del usuario, pero no afecta directamente a los archivos del cliente, solamnente se ejecuta en el navegador.
- Se ejecuta ni bien es cargado en el arachivo HTML.

:::tip
Piensa en javascript como el cerebro de tu sitio web, que se encarga de la interaccion con el usuario. Añadiendo dinamismo y vida a tu sitio web.
Es importante entender que javascript no puede interactuar directamente con nuestro equipo.
:::

## Comenzando a escribir Javascript `<script>`

Para escribir y ejecutar codigo Javascript en un archivo HTML, se utiliza la etiqueta `<script>`, que se coloca en el `<head>` o al final del  `<body>` del archivo HTML.
La etiqueta `<script>` es una etiqueta con contenido(Codigo), ahora escribe tu primer *Hola Mundo* en Javascript.
- La funcion `alert()` muestra un mensaje en una ventana emergente.
```html
<script>
    alert("Hola Mundo");
    // alert recibe un String como parametro
    alert("Bienvenido a mi sitio web");
</script>
```

:::caution
Nota que el mensaje `Hola Mundo` se muestra una vez la pagina es cargada, esto se debe a que el codigo Javascript **se ejecuta ni bien es cargado** en el archivo HTML.
La funcion `alert()` no es recomendada para mostrar mensajes en un sitio web, ya que puede llegar a ser molesto para el usuario.
:::



## Consola de Javascript
Usar la funcion `alert()` puede llegar a ser molesto, lo mas recomendable es usar la **consola** de Javascript para ver los mensajes de depuracion de nuestro codigo.

La consola de Javascript es una herramienta que nos permite ver los mensajes de error, advertencias y mensajes de depuracion de nuestro codigo Javascript.

```javascript
// Imprimir un mensaje en la consola
var edad =10;
let nombre = "Juan";
// Asignacion de variables
edad = 15
console.log("Hola, mi nombre es "+nombre+" y tengo "+edad+" años");
```

:::tip
Para abrir la consola de Javascript en tu navegador, presiona `F12` y selecciona la pestaña `Consola`.
:::

## Cargando Javascript en un archivo externo
El codigo Javascript puede ser escrito en un archivo aparte y ser incluido en el archivo HTML con la propiedad `src` de la etiqueta `<script>`.


```html "src="scripts/hola.js""
<!--scripts/hola.js-->
<!-- Incluyendo un script externo -->
<script src="scripts/hola.js"></script>
```
:::nota
Los scripts tiene la terminacion `.js`, y se recomienda que se encuentren en la carpeta `scripts` de tu proyecto.

Tambien recuerda que el codigo Javascript se ejecuta ni bien es cargado en el archivo HTML.
:::

## Variables
:::warning
Debe realizar todo el codigo de esta seccion en un archivo js para su revision
:::

Javascript es un lenguaje de **tipado dinamico**, lo que significa que no es necesario especificar el tipo de dato que se almacenara en una variable, ya que el lenguaje lo hace por nosotros.

Existen tres formas de declarar variables en javascript:
- `var`: es la forma antigua de declarar variables, pero aun se utiliza en algunos casos.
- `let`: es la forma moderna de declarar variables, y es la mas recomendada.

```javascript
// Declaracion de variables en javascript
var edad = 10;
let nombre = "Juan";

alert(nombre);
alert(edad);
```

### Operaciones de incremento y decremento.
Tambien conocidos como **operadores unarios**, se utilizan para incrementar o decrementar el valor de una variable en una unidad.
```js
// Incremento y decremento (despues)
a++;
b--;
console.log(a);
console.log(b);
// Incremento y decremento (antes)
++a;
--b;
console.log(a);
console.log(b);
```

### Operadores de comparacion
Los operadores de comparacion se utilizan para comparar dos valores y retornar un valor booleano `true` o `false`.
```js
// Operadores de comparacion
a > b; // true
a < b; // false
a == b; // false
a != b; // true
```

### Operadores logicos
Los operadores logicos se utilizan para realizar operaciones logicas entre dos valores booleanos.
```js
// Operadores logicos
true || false // true
true && false // false
!true // false
```

## Tipos de datos
Javascript es un lenguaje de tipado dinamico, lo que significa que no es necesario especificar el tipo de dato que se almacenara en una variable, ya que el lenguaje lo hace por nosotros.

Esto llega a ser una ventaja y desventaja:
- Ventaja: no es necesario especificar el tipo de dato.
- Desventaja: puede llegar a ser confuso el tipo de dato que se esta almacenando en una variable.


Note que en el siguiente ejemplo, la variable `a` es un numero, pero al sumarla con la variable `b` que es un string, el resultado es un string, y el resultado de la **suma** no es el esperado.
```js
// Declaracion de variables
let a = '1'
let b = 7
let suma = a + b
console.log(suma) // 17
```

### La funcion `typeof`
La funcion `typeof` nos permite saber el tipo de dato que se esta almacenando en una variable.

```js
// Declaracion de variables
let a = 10;
let b = "Hola Mundo";
let c = true;

console.log(typeof a); // number
console.log(typeof b); // string
console.log(typeof c); // boolean
```
:::note
`tipeof` retorna un string con el tipo de dato que se esta almacenando en una variable, esto nos permite hacer comparaciones y validaciones en nuestro codigo.
:::


## Las Funciones
Las funciones son bloques de codigo que se pueden reutilizar en nuestro programa, se declaran con la palabra reservada `function`.


###  Definicion de funciones
Las funciones son bloques de codigo que se pueden reutilizar en nuestro programa, se declaran con la palabra reservada `function`.

- Se declaran con la palabra reservada `function`.
- Se le asigna un nombre a la funcion con terminado de parentesis `()`.
- Se le asigna un bloque de codigo entre llaves `{}`.

```js
// Declaracion de una funcion
function saludar(){
    console.log("Hola Mundo");
    alert("Revise la consola");
}
```

### Llamada a una funcion
Para llamar a una funcion, se utiliza el nombre de la funcion seguido de parentesis `()`.

Esto ejecutara el bloque de codigo que se encuentra en la funcion.

```js ins={7}
function saludar(){
    // Bloque de codigo
    console.log("Hola Mundo");
    alert("Revise la consola");
}
// Llamada a la funcion
saludar();
```

### Funciones anonimas
Las funcioens aninomas carecen de nombre, y se las asigna a una variable o a una propiedad de un objeto.
```js
// Funcion anonima
function(){
    console.log("Hola Mundo");
}
```

Estas funciones por si solas no se ejecutan, se deben asignar a una constante o variable para que almacenar la funcion y luego ejecutarla.

```js
// Uso correcto de una funcion anonima
const saludar = function(){
    console.log("Hola Mundo");
}
// Ejeccucion de la funcion
saludar();
```

#### Usando funciones como si fueran variables
Se puede reasignar la funcion a otra variable, y se ejecutara la nueva funcion asignada.
```js
// Uso en variables
let miFuncion = function(){
    console.log("Hola Mundo");
    alert("Revise la consola");
}
miFuncion(); // Ejeccucion de la funcion

// Reasingar la funcion
miFuncion = function(){
    console.log("La funcion ha sido reasignada");
    console.log("Adios Mundo");
}
miFuncion();

```

### Funciones como referencia
Las funciones en Javascript son tratadas como cualquier otro tipo de dato, se las puede asignar a una variable como si fuera un numero o un string, per sin el uso de parentesis `()`.

```js
// Declaracion de una funcion
function miFuncion(){
    console.log("Hola Mundo");
}

// Asignacion de una funcion a una constante
const hola = miFuncion;
hola(); // Ejecucion de la funcion
```

:::note
Cuando se usa la funcion sin parentesis `()`, se esta asignando la funcion a la variable, y **no se esta ejecutando** la funcion.

Imaginelo como si estuviera guardando la funcion en una caja, y luego la ejecutara cuando lo necesite.
:::

## Los Objetos
Los objetos son estructuras de datos que nos permiten almacenar multiples valores en una sola variable, se declaran con llaves `{}`.La forma de indicar la clave y el valor es `clave: valor`.

Cada par de clave y valor se separa por una coma `,`.

```js
const persona = {
    nombre: "Juan",
    apellido: "Perez",
    edad: 25,
    casado: false
}
```

### Acceder valores de un objeto
Para acceder a los valores de un objeto, se utiliza la notacion de punto `.` o la notacion de corchetes `[]`.

```js
// Acceder a los valores de un objeto
console.log(persona.nombre); // Juan
console.log(persona["apellido"]); // Perez
console.log(persona.edad); // 25
console.log(persona.casado); // false
```

:::note
El uso de corchetes `[]` es util cuando se quiere acceder a una propiedad de un objeto, pero no se conoce el nombre de la propiedad, o que la clave tenga caracteres especiales como espacios o guiones.
```js
const persona = {
    "nombre completo": "Juan Perez"
}

console.log(persona["nombre completo"]); // Juan Perez
```
:::

### Modificar valores de un objeto
Para modificar los valores de un objeto se las trata como cualquier otra variable, se asigna un nuevo valor a la propiedad del objeto.
```js
// Modificar valores de un objeto
persona.edad = 30;
persona.casado = false;
console.log(persona.edad); // 30
console.log(persona.casado); // false
```


### Funciones dentro de  Objetos(Metodos)
Los objetos pueden tener funciones como valores de sus propiedades, a estas funciones se les llama **metodos**.

:::note
Para asignar una funcion a una propiedad de un objeto, se recomienda usar las  **funciones anonimas**.
:::

```js
const persona = {
    nombre: "Juan",
    apellido: "Perez",
    edad: 25,
    casado: false,
    saludar: function(){
        console.log("Hola Mundo");
    }
}
// Llamada a un metodo
persona.saludar();
```
### Definicino clave `this` en objetos
La palabra reservada `this` se refiere al objeto que contiene la funcion, es decir, que `this` hace referencia al objeto en el que se encuentra la funcion.

Tambien podemos modificar las propiedades del objeto con la palabra reservada `this`.

```js
const persona = {
    nombre: "Juan",
    apellido: "Perez",
    edad: 25,
    casado: false,
    saludar: function(){
        console.log("Hola, mi nombre es "+this.nombre+" "+this.apellido);
    },
    matrimonio: function(){
        this.casado = true;
        console.log("Ahora estoy casado");
    }
}

// Llamada a un metodo
persona.saludar();
persona.matrimonio();
console.log(persona.casado); // Ya esta casado
```

### Objetos anidados
Los objetos pueden tener otros objetos como valores de sus propiedades, a estos objetos se les llama **objetos anidados**.

```js
const persona = {
    nombre: "Juan",
    apellido: "Perez",
    edad: 25,
    casado: false,
    direccion: {
        calle: "Av. 6 de Diciembre",
        numero: 123,
        ciudad: "Quito"
    }
}
```

Para acceder a los valores de un objeto anidado, se utiliza la notacion de punto `.` o la notacion de corchetes `[]`.

```js
// Acceder a los valores de un objeto anidado
console.log(persona.direccion.calle); // Av. 6 de Diciembre
console.log(persona.direccion.numero); // 123
console.log(persona["direccion"]["ciudad"]); // Quito
```

Como pudo ver en javascript **casi todo** se trata como un objeto, es importante entender esto para la siguiente seccion.

## Eventos en Javascript
Los eventos son acciones que ocurren en la pagina web, como hacer **click** en un boton, enviar un formulario, cargar la pagina, etc.

La manera antigua de asignar eventos a los elementos del DOM es con la propiedad `onclick` y asignarle una funcion que se ejecutara cuando ocurra el evento.

<Tabs>
<TabItem label="index.html">
```html "onclick="saludar()""
<button type="submit" id="btn-form" onclick="saludar()">Iniciar sesión</button>
```
</TabItem>

<TabItem label="script.js">
```js
function saludar() {
  alert("Hola Mundo");
}
```
</TabItem>
</Tabs>
Ahora cuando se haga click en el boton `Iniciar sesion`, se ejecutara la funcion `saludar()`.

### Manejo de eventos con buenas practicas
Cuando se realiza una pagina web, es recomendable separar el codigo Javascript del archivo HTML, para que sea mas facil de mantener y de entender. Por lo que no debe evitar usar codigo Javascript en el archivo HTML.

La alternativa a la [seccion anterior](/guides/05-javascript/#eventos-en-javascript) es asignar un evento a un elemento del DOM con la funcion `addEventListener()`, que recibe dos parametros:
1. El nombre del evento. Ej. `click`, `submit`, `load`, etc.
2. La funcion que se ejecutara cuando ocurra el evento. 
    Esta debe ser una referencia o una funcion anonima.

```js
// Seleccionar un elemento del DOM (boton)
const boton = document.querySelector("#btn-form");

// Definimos la funcion a usar
function saludar() {
  alert("Hola Mundo");
}

// Asignar un evento a un elemento del DOM
boton.addEventListener("click", saludar);
```
Esto deberia hacer lo mismo que el ejemplo anterior, pero con la diferencia que el codigo Javascript esta separado del archivo HTML.


### Otros eventos comunes
Todos los eventos se definen de la misma manera, solo cambia el nombre del evento que se quiere asignar. Se le puede aplicar a las distintas etiquetas HTML estudiadas.

Ej: El objeto `window` hacer referencia a la ventana del navegador, y se puede asignar eventos a la ventana del navegador.
```js
window.addEventListener("load", function(){
    alert("La pagina ha sido cargada");
});
```
Algunos de los eventos mas comunes son:

| Evento | Descripcion |Etiqutas HTML |
|--------|-------------|--------------|
| click  | Se ejecuta cuando se hace click en un elemento | Todos los elementos |
| submit | Se ejecuta cuando se envia un formulario | Formularios |
| load   | Se ejecuta cuando la pagina ha sido cargada | Ventana del navegador |
| keyup  | Se ejecuta cuando se suelta una tecla | Input, textarea |
| change | Se ejecuta cuando se cambia el valor de un input | Input, select, textarea |
| focus  | Se ejecuta cuando un elemento recibe el foco | Todos los elementos |
| blur   | Se ejecuta cuando un elemento pierde el foco | Todos los elementos |
| scroll | Se ejecuta cuando se hace scroll en la pagina | Ventana del navegador |
|mouseover| Se ejecuta cuando el cursor pasa por encima de un elemento | Todos los elementos |
|mouseout | Se ejecuta cuando el cursor sale de un elemento | Todos los elementos |

Puede consultar la [documentacion de eventos](https://developer.mozilla.org/es/docs/Web/Events) para mas informacion sobre los eventos en Javascript.



## Condicionales `if` y `else`
Los condicionales son estructuras de control que que en base a una condicion, ejecutan un bloque de codigo u otro.

Al igual que otros lenguajes se tiene las palabras reservadas:
- `if`: si la condicion es verdadera, se ejecuta el bloque de codigo.
- `else`: si la condicion es falsa, se ejecuta el bloque de codigo.

```js
// Declaracion de variables
let a = 10;
let b = 5;

// Condicional
if(a > b){
    // Si la condicion es verdadera
    console.log("a es mayor que b");
}
else{
    // Si la condicion es falsa
    console.log("a es menor que b");
}
```
